# Building a Budget Proxmox Home Server

Proxmox Virtual Environment (VE) has become the go-to choice for homelab enthusiasts looking to virtualize their home infrastructure. It's free, open-source, and incredibly powerful. In this guide, I'll walk you through building a budget-friendly Proxmox home server that won't break the bank but will give you enterprise-grade virtualization capabilities at home.

![Server Room](/images/builds/budget-proxmox-home-server-build/server-room.jpg)

## Why Proxmox for Your Homelab?

Proxmox VE combines two virtualization technologies in one platform: KVM (Kernel-based Virtual Machine) for virtual machines and LXC (Linux Containers) for lightweight containerization. This dual approach gives you flexibility that's hard to match with other solutions.

<Callout type="tip" title="Proxmox vs ESXi">
Proxmox offers several advantages over VMware ESXi for homelab use: it's completely free with no locked features, has built-in backup tools, supports ZFS natively, and has an active community. While ESXi is excellent, its free tier is limited, and Broadcom's acquisition has raised concerns about future licensing.
</Callout>

### Key Benefits

- **No licensing costs**: Full features available without paying
- **Web-based management**: Manage everything from your browser
- **Built-in clustering**: Scale to multiple nodes when ready
- **ZFS support**: Enterprise-grade storage management
- **Proxmox Backup Server**: Free backup solution that integrates seamlessly
- **Active community**: Help is always available on forums

### CPU Considerations

The Intel Core i5-13500 is our sweet spot for several reasons:

1. **14 cores total**: 6 performance cores + 8 efficiency cores means excellent multi-tasking
2. **Low power idle**: Draws only 10-15W at idle with proper configuration
3. **QuickSync**: Built-in video transcoding for media servers like Plex or Jellyfin
4. **ECC support**: Works with consumer ECC memory if you can find it

AMD alternatives like the Ryzen 7 5700G are also excellent choices, offering integrated graphics and 8 full performance cores. The main tradeoff is slightly higher idle power consumption.

### Memory: Never Enough RAM

Virtualization is RAM-hungry. A good rule of thumb is:

- **Minimum**: 16GB for basic setup
- **Recommended**: 32GB for comfortable operation
- **Ideal**: 64GB if running memory-intensive workloads

<Callout type="info" title="Memory Overcommitment">
Proxmox allows memory overcommitment for VMs, meaning you can allocate more RAM than physically available. However, this only works well when VMs don't all need their full allocation simultaneously. For LXC containers, memory is more efficiently shared via KSM (Kernel Samepage Merging).
</Callout>

### Storage Strategy

Your storage choices significantly impact performance and budget:

**Boot/VM Storage (NVMe SSD)**
A 1TB NVMe SSD gives you:
- Fast boot times
- Snappy VM performance
- Room for multiple test environments
- Proxmox's default LVM-thin storage

**Bulk Storage (HDD)**
A single 4TB NAS drive provides:
- Media storage
- Backup destination
- ISO repository

## Installation Walkthrough

### Preparing Your Boot Media

1. Download the Proxmox VE ISO from the official website
2. Flash it to a USB drive using Ventoy or balenaEtcher

```bash
# Using dd on Linux (replace /dev/sdX with your USB device)
sudo dd bs=1M conv=fsync oflag=direct status=progress if=proxmox-ve_8.1-2.iso of=/dev/sdX

# Or use Etcher/Ventoy for a GUI approach
```

### BIOS Configuration

Before installing, configure your BIOS properly:

1. **Enable virtualization**: Intel VT-x or AMD-V
2. **Enable IOMMU**: Required for PCIe passthrough
3. **Set power profile**: Balanced or power saver for lower idle draw
4. **Disable unused devices**: Serial ports, unused SATA controllers

### Installation Steps

1. Boot from USB and select "Install Proxmox VE"
2. Accept the EULA
3. Select your target disk (the NVMe SSD)
4. Set country, time zone, and keyboard layout
5. Enter a strong root password and email for notifications
6. Configure network settings:
   - Static IP is recommended
   - Set your gateway and DNS servers
7. Complete installation and reboot

## Post-Installation Configuration

After your first boot, there are several tasks to optimize your setup:

### Update Your System

```bash
# Update package repositories and system
apt update && apt dist-upgrade -y

# Install commonly needed packages
apt install -y curl wget git htop iotop
```

### Remove Enterprise Repository

<Callout type="tip" title="Free Repository">
By default, Proxmox uses the enterprise repository which requires a subscription. For homelab use, switch to the no-subscription repository for updates.
</Callout>

```bash
# Remove enterprise repository
rm /etc/apt/sources.list.d/pve-enterprise.list

# Add no-subscription repository for Proxmox 8.x
cat <<EOF > /etc/apt/sources.list.d/pve-no-subscription.list
deb http://download.proxmox.com/debian/pve bookworm pve-no-subscription
EOF

# Update again
apt update
```

### Disable the Nag Screen

```bash
# Disable the no-subscription notification
sed -i "s/data.status !== 'Active'/false/g" /usr/share/javascript/proxmox-widget-toolkit/proxmoxlib.js
```

### Configure Power Management

For an efficient home server:

```bash
# Install power management tools
apt install -y powertop

# Tune power settings
powertop --auto-tune

# Enable powertop auto-tune on boot
systemctl enable powertop
```

## Performance Tuning Tips

### CPU Governor

```bash
# Set to powersave for lower idle consumption
echo powersave | tee /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor
```

### ZFS ARC Size

If using ZFS (recommended for data integrity):

```bash
# Limit ARC to 25% of RAM for better VM performance
echo "options zfs zfs_arc_max=$(awk '/MemTotal/ {printf "%d", $2*1024*0.25}' /proc/meminfo)" >> /etc/modprobe.d/zfs.conf
```

### Network Tuning

```bash
# Enable jumbo frames if your network supports it
ip link set eth0 mtu 9000

# Make persistent by adding to /network/interfaces
```

## Troubleshooting Common Issues

<Callout type="info" title="Community Resources">
The Proxmox forums and subreddit (r/Proxmox, r/homelab) are invaluable resources. Most issues you encounter have been solved and documented by the community.
</Callout>

### High Power Consumption

If your server draws more than expected at idle:

1. Check BIOS power settings
2. Disable unused hardware in BIOS
3. Run `powertop` to identify power hogs
4. Consider undervolting if comfortable with it

### VM Boot Failures

1. Verify virtualization is enabled in BIOS
2. Check disk space availability
3. Ensure the VM OS is compatible with VirtIO drivers

### Network Issues

1. Verify bridge configuration in `/etc/network/interfaces`
2. Check that `vmbr0` is present and active
3. Ensure your router isn't blocking the Proxmox host
