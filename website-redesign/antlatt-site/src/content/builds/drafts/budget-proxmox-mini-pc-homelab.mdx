## Introduction

Building a homelab doesn't have to break the bank. Mini PCs have become incredibly popular for homelab enthusiasts who want to run virtualization platforms like Proxmox VE without the noise, heat, and power consumption of traditional rack servers. In this guide, we'll walk through setting up a budget-friendly Proxmox homelab using readily available mini PCs.

Mini PCs offer several advantages for homelab builders: they're compact, energy-efficient, surprisingly powerful, and often available at faction of the cost of enterprise hardware. Whether you want to run Docker containers, Home Assistant, Plex, or experiment with Kubernetes, a mini PC Proxmox cluster can handle it all.

## Step 1: Choose Your Mini PC Hardware

The heart of your Proxmox homelab is the mini PC itself. For budget builds, Intel N100 or N5105 processors offer excellent value with 4 cores and low power consumption. The Beelink S12 Pro and EQ12 are popular choices in the homelab community.

**What to look for:**
- Intel N100/N5105 or better CPU (avoid Atom/Celeron if possible)
- At least 8GB RAM (16GB recommended)
- NVMe storage slot (SATA SSD works but NVMe is faster)
- At least one 2.5GbE Ethernet port
- VT-x and VT-d support for virtualization

**Popular Mini PC Options:**
- **Budget:** Beelink S12 Pro ($180-200) - Intel N100, great value
- **Mid-range:** ASUS NUC 14 Pro ($350+) - More powerful, better build
- **Storage focus:** Aoostar WTR Pro ($250+) - Has dual 2.5" drive bays

<Callout type="tip">
Before buying, check if the mini PC has an empty NVMe slot or if you need to replace the existing drive. Some models make storage upgrades easier than others.
</Callout>

## Step 2: Download and Create Proxmox Installer

Proxmox VE is a free, open-source virtualization platform based on Debian Linux. It provides a web-based interface for managing VMs, containers, and storage.

1. Download Proxmox VE ISO from the official website: https://www.proxmox.com/downloads
2. Use Ventoy or Rufus to create a bootable USB drive
3. Insert the USB drive into your mini PC and boot from it

**Creating the bootable USB with Ventoy:**
```bash
# Install Ventoy on your USB drive
sudo ventoy -i /dev/sdX

# Copy the Proxmox ISO directly to the Ventoy partition
cp proxmox-ve_8.1.iso /media/your-usb/
```

## Step 3: Install Proxmox VE

Installing Proxmox is straightforward. Connect a monitor and keyboard to your mini PC for the initial setup.

1. Boot from the USB drive (usually F7 or F12 for boot menu)
2. Select "Install Proxmox VE" from the boot menu
3. Accept the license agreement
4. Select your target disk (the NVMe drive)
5. Set your country and timezone
6. Create a password and email for the admin user
7. Configure network settings (use static IP if possible)
8. Complete the installation and reboot

After installation, access Proxmox via your browser at `https://YOUR-IP:8006`

<Callout type="tip">
Use a static IP address outside your DHCP range to avoid conflicts. This makes it easier to manage your homelab services long-term.
</Callout>

## Step 4: Post-Installation Configuration

Once Proxmox is running, there are essential configurations to perform:

**Update the system:**
```bash
apt update && apt dist-upgrade -y
```

**Remove the enterprise repository (for free version):**
```bash
rm /etc/apt/sources.list.d/pve-enterprise.list
echo "deb http://download.proxmox.com/debian/pve bookworm pve-no-subscription" > /etc/apt/sources.list.d/pve-no-subscription.list
```

**Install useful tools:**
```bash
apt install -y iftop htop ncdu
```

## Step 5: Create Your First Virtual Machine

Now comes the fun part - running your first VM!

1. Download an ISO (Ubuntu Server, Debian, or whatever you prefer)
2. Upload it to Proxmox via the web interface (select your node > local-lvm > ISO Images)
3. Click "Create VM" in the top right
4. Configure CPU, RAM, and storage allocation
5. Start the VM and begin your homelab journey!

**Recommended starter VMs:**
- **Ubuntu Server:** For Docker containers and self-hosted apps
- **Home Assistant OS:** For smart home automation
- **Pi-hole:** For network-wide ad blocking
- **Plex/Jellyfin:** For media streaming
