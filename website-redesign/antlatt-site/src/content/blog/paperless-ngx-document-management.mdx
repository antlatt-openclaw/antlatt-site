## Hardware Requirements

Paperless-ngx is surprisingly lightweight. OCR processing is the main resource consumer—a multi-page PDF with high-resolution scans will briefly spike CPU and memory usage during processing.

### Minimum Requirements

| Component | Minimum | Recommended |
|-----------|---------|-------------|
| CPU | 2 cores | 4+ cores |
| RAM | 2GB | 8GB+ |
| Storage | 50GB | 500GB+ SSD |
| OS | Linux (Docker) | Any Docker host |

:::tip[Memory Matters for OCR]
If you plan to process many documents daily or handle high-resolution scans, 8GB RAM is the sweet spot. Tesseract OCR is memory-hungry—insufficient RAM will slow processing significantly.
:::

### Storage Planning

Plan for growth. A typical scanned document averages 1-2MB. Consider:

- **Light use** (hobbyist): 50-100GB provides years of headroom
- **Family archive** (taxes, medical, everything): 250-500GB
- **Power user** (scanning everything, high-res photos): 1TB+

SSD storage is strongly recommended for fast search indexing.

## Quick Installation with Docker

The fastest way to get Paperless-ngx running is Docker Compose. This method works on any Linux server, Proxmox VM, or even a Raspberry Pi (with 4GB+ RAM).

### Step 1: Create the Directory Structure

```bash
mkdir -p ~/paperless-ngx/{data,media,consume,export}
cd ~/paperless-ngx
```

### Step 2: Create docker-compose.yml

```yaml
version: "3.8"
services:
  paperless:
    image: ghcr.io/paperless-ngx/paperless-ngx:latest
    container_name: paperless
    restart: unless-stopped
    environment:
      - PAPERLESS_URL=https://paperless.yourdomain.com
      - PAPERLESS_SECRET_KEY=your-secret-key-change-this
      - PAPERLESS_ADMIN_USER=admin
      - PAPERLESS_ADMIN_PASSWORD=your-secure-password
      - PAPERLESS_TIME_ZONE=America/New_York
      - PAPERLESS_OCR_LANGUAGE=eng
      - PAPERLESS_CONSUMER_POLLING=60
    volumes:
      - ./data:/usr/src/paperless/data
      - ./media:/usr/src/paperless/media
      - ./consume:/usr/src/paperless/consume
      - ./export:/usr/src/paperless/export
    ports:
      - "8000:8000"
```

### Step 3: Start Paperless-ngx

```bash
docker compose up -d
docker compose logs -f  # Watch the startup logs
```

First startup takes a few minutes as it initializes the database and creates the search index. Once you see `Paperless-ngx is ready`, navigate to `http://your-server:8000` and log in with the credentials you defined.

:::note[Environment Variables]
Key settings to customize:
- **PAPERLESS_URL**: Your external URL (required for email links and proper CSP headers)
- **PAPERLESS_SECRET_KEY**: Random string for session security (generate with `openssl rand -hex 32`)
- **PAPERLESS_OCR_LANGUAGE**: Set to your primary language; add multiples like `eng+fra` for French documents
:::

### Behind a Reverse Proxy

For production use, always run behind HTTPS. Traefik and Caddy are popular choices:

```yaml
# Example with Caddy labels
services:
  paperless:
    # ... existing config ...
    labels:
      - "caddy=paperless.yourdomain.com"
      - "caddy.reverse_proxy={{upstreams 8000}}"
```

## Understanding the Directory Structure

Paperless-ngx uses four key directories that serve specific purposes:

| Directory | Purpose | Contents |
|-----------|---------|----------|
| **consume** | Upload directory | Drop files here for auto-import |
| **media** | Document storage | Original files + processed versions |
| **data** | Database and index | SQLite database, search index |
| **export** | Backup location | Exported documents for backup |

The **consume** folder is your inbox. Any file dropped here—PDF, image, even email files—gets automatically processed, OCR'd, and added to your library. This enables powerful automation: set up a network share, configure your scanner to output there, and documents appear in Paperless automatically.

## Getting Documents In

Paperless-ngx offers multiple import methods. The right approach depends on your workflow.

### Method 1: Consume Folder (Recommended)

The simplest and most automated approach. Configure your scanner or phone app to save files directly to the consume directory.

```bash
# Create a network share for easy scanner access
sudo apt install samba
# Add to /etc/samba/smb.conf:
[paperless-consume]
   path = /home/youruser/paperless-ngx/consume
   browseable = yes
   read only = no
```

Now your scanner can save directly to Paperless—no manual intervention required.

### Method 2: Web Interface Upload

For occasional documents, use the web UI:

1. Click the upload button (or drag and drop)
2. Select files from your computer
3. Assign tags, correspondents, and document type during upload
4. Submit for processing

This works well for manual entry where you want immediate metadata control.

### Method 3: Email Processing

Perfect for digital statements, receipts, and anything that arrives in your inbox:

```yaml
# Add to docker-compose environment
- PAPERLESS_CONSUMER_ENABLE_ASN_BARCODE=true
- PAPERLESS_EMAIL_ENABLE=true
- PAPERLESS_EMAIL_HOST=imap.gmail.com
- PAPERLESS_EMAIL_PORT=993
- PAPERLESS_EMAIL_USER=your-email@gmail.com
- PAPERLESS_EMAIL_PASSWORD=your-app-password
- PAPERLESS_EMAIL_USE_SSL=true
```

Paperless can monitor multiple email folders, auto-process attachments, and even apply tags based on sender or subject patterns.

### Method 4: Mobile Apps

Third-party apps extend Paperless to your phone:

- **Paperless-Share (iOS)**: Share extension for quick scanning and upload
- **Docpile (Android)**: Camera scanning with direct Paperless integration
- **Generic approach**: Use any scanner app that exports to WebDAV or a cloud folder synced to your consume directory

## OCR and Search: Finding Anything Instantly

This is where Paperless-ngx shines. Every document gets processed through Tesseract OCR, making all text—including text inside images—searchable.

### How OCR Works

When you upload a document:

1. Paperless detects the file format
2. If an image or scanned PDF, it runs OCR with your configured language
3. All extracted text is indexed by the search engine
4. The document becomes searchable by any word it contains

Search for "insurance policy deductible" and Paperless finds every document mentioning those terms—even if they're inside a scanned PDF.

### Configuring OCR Languages

```yaml
# Multiple languages
- PAPERLESS_OCR_LANGUAGE=eng+spa+fra

# Skip OCR for already-searchable PDFs (faster processing)
- PAPERLESS_OCR_SKIP_ARCHIVE_FILE=never
```

:::tip[Language Detection]
Paperless can auto-detect languages, but specifying them improves accuracy. Add all languages you expect to encounter.
:::

### Search Syntax

The search bar supports powerful queries:

| Query | Result |
|-------|--------|
| `invoice amazon` | Documents with both terms |
| `created:[2026-01-01 TO 2026-12-31]` | Date range search |
| `tag:receipt` | Specific tag |
| `correspondent:IRS` | Specific sender |
| `content:"warranty expires"` | Exact phrase |

Combine filters for precision: `tag:receipt correspondent:Costco created:[2026-01-01 TO 2026-03-31]` finds all Costco receipts from Q1.

## Automatic Tagging and Organization

The real power of Paperless-ngx is automation. Set it up once, and it learns your preferences.

### Rules-Based Auto-Tagging

Create matching rules that apply tags automatically:

```yaml
# Example: Auto-tag Amazon purchases
# Settings > Mail & Tags > Tags > Create
Tag: "amazon"
Matching algorithm: "Contains" 
Match: "amazon.com"
```

Every document containing "amazon.com" now gets the "amazon" tag automatically.

### Machine Learning Suggestions

Paperless learns from your behavior. When you manually apply tags, it builds a model and starts suggesting tags for new documents. The more you use it, the smarter it gets.

```yaml
# Enable learning (default is on)
- PAPERLESS_CLASSIFIER_ENABLE=true
```

:::note[Training Tips]
- Be consistent with tagging early on
- Apply multiple tags when appropriate
- Correct wrong suggestions—the model learns from corrections
:::

**Correspondents** track who sent/received it:

## Workflow Automations

Paperless-ngx workflows automate complex processing pipelines.

### Example 1: Receipt Monthly Organization

```yaml
# Workflow: Receipt Processing
# Settings > Workflows > New

Trigger: Document added with tag "receipt"
Actions:
  1. Set document type: Receipt
  2. If correspondent contains "Amazon", assign tag "online"
  3. If created date is in January, add tag "2026-01"
  4. Assign storage path: /receipts/{created_year}/{created_month}/
```

Receipts automatically file themselves into monthly folders.

### Example 2: Tax Document Pipeline

```yaml
# Workflow: Tax Document Processing

Trigger: Document added with tag "tax"
Actions:
  1. Set document type: Tax Document
  2. Add tag corresponding to tax year (auto-extract from date)
  3. Create saved view link for "Tax 2026"
```

At tax time, export the "Tax 2026" saved view and you have everything organized.

## Security Best Practices

Your document archive contains sensitive information. Treat it accordingly.

### Authentication Hardening

```yaml
# Strong security settings
- PAPERLESS_DISABLE_REGULAR_LOGIN=false  # Keep password login
- PAPERLESS_ENABLE_HTTP_REMOTE_USER=true  # Enable SSO proxy auth
```

For family deployments, create separate user accounts with appropriate permissions:

```yaml
# User permissions levels
- Owner: Full access, can manage users
- Editor: Can add and modify documents
- Viewer: Read-only access
```

### Backup Strategy

Paperless data is irreplaceable. Follow the 3-2-1 rule: three copies, two different media types, one off-site.

```bash
#!/bin/bash
# paperless-backup.sh - Run daily via cron

DATE=$(date +%Y%m%d)
BACKUP_DIR="/backup/paperless"

# Export documents and database
docker exec paperless document_exporter /export

# Create archive
tar -czf $BACKUP_DIR/paperless-$DATE.tar.gz \
  /path/to/paperless/media \
  /path/to/paperless/data \
  /path/to/paperless/export

# Keep last 30 days locally
find $BACKUP_DIR -name "paperless-*.tar.gz" -mtime +30 -delete

# Sync to off-site (rclone to cloud storage)
rclone sync $BACKUP_DIR remote:paperless-backups/
```

### Network Isolation

Don't expose Paperless directly to the internet. Recommended architecture:

1. Paperless runs on internal network only
2. VPN (Tailscale/WireGuard) for remote access
3. If public access needed, use reverse proxy with:
   - Let's Encrypt TLS certificate
   - Rate limiting
   - Strong authentication (ideally SSO)
   - Optionally: IP allowlisting

## Migrating from Physical Archives

Ready to digitize your existing paper collection? Here's a practical approach.

### Scanner Recommendations

| Budget | Model | Notes |
|--------|-------|-------|
| Under $150 | Brother ADS-2700W | WiFi, duplex, reliable |
| Mid-range | Fujitsu ScanSnap iX1600 | Excellent software, fast |
| Professional | Fujitsu fi-8000 series | 80ppm, heavy duty |

:::tip[Versus Flatbed Scanners]
Sheet-fed scanners (like ScanSnap) are dramatically faster than flatbed for documents. Keep a flatbed for photos, books, and fragile items, but use sheet-fed for bulk digitization.
:::

### Scanning Best Practices

1. **Resolution**: 300 DPI is sufficient for text documents
2. **Color mode**: Auto-detect saves space—black and white documents don't need color
3. **Duplex**: Scan both sides—Paperless will ignore blank pages
4. **File format**: PDF is ideal; Paperless also handles JPG, PNG, TIFF
5. **Batch scanning**: Scan multiple documents; separate with blank pages if your scanner supports it

### Digitization Workflow

1. **Sort by type**: Group similar documents before scanning
2. **Batch scan**: Process each group in one session
3. **Quick review**: Check that OCR worked (sample documents)
4. **Add metadata**: Apply tags, fix auto-detected correspondents
5. **Shred or store**: Securely dispose of or archive originals

:::note[What to Keep Physical]
Some documents should remain physical:
- Passports, driver's licenses (keep original)
- Original signed contracts (may be legally required)
- Birth certificates, marriage licenses
- Vehicle titles
- Professional licenses

Digitize these for convenience, but store originals securely.
:::

## Troubleshooting Common Issues

### OCR Not Working

**Symptom**: Documents don't become searchable

**Solutions**:
1. Check OCR language configuration: `PAPERLESS_OCR_LANGUAGE=eng`
2. Verify sufficient RAM (OCR fails silently on low memory)
3. Check logs: `docker logs paperless | grep -i ocr`

### Slow Processing

**Symptom**: Documents take minutes to process

**Solutions**:
1. Increase container memory limit
2. Enable parallel processing: `PAPERLESS_TASK_WORKERS=2`
3. Use SSD storage for media directory

### Search Not Finding Documents

**Symptom**: Known text doesn't appear in search

**Solutions**:
1. Rebuild search index: `docker exec paperless document_index reindex`
2. Check that OCR actually extracted text
3. Verify your search syntax
